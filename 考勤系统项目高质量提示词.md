# 考勤系统项目高质量提示词

本提示词用于指导 AI 完成面向中小企业的轻量级考勤数据管理系统开发，分两个阶段推进，严格遵循以下结构定义开发要求：

---

## 第一阶段：核心功能版（MVP）

### 1. 角色与上下文定义

你现在是一名资深的前端开发工程师，正在开发一款面向中小企业的轻量级考勤数据处理工具。当前处于项目 MVP 开发阶段，采用纯前端单页应用架构，利用浏览器本地存储实现无需部署的即时使用，目标是替代手工 Excel 计算，实现复杂考勤规则的自动化判定，为企业财务发薪提供准确凭证。

### 2. 清晰的任务目标

开发纯前端架构的考勤管理 MVP 版本，实现多源考勤数据的自动导入识别、复杂考勤规则自动化计算、考勤数据可视化查询与可配置化导出，满足企业财务考勤数据归档的核心需求。

### 3. 具体的输入输出格式

#### 输入

1. 用户输入：管理员账号密码（前端形式验证）、考勤规则配置参数、公司节假日 / 调休配置、批量 Excel 考勤源文件

2. 源文件类型：

    - 指纹考勤机原始 / 汇总数据 Excel

    - OA 请假 / 加班 / 漏打卡说明 / 出差申请单 Excel

    - 员工排班表 Excel

#### 输出

1. 交互界面输出：

    - 数据导入界面：支持多文件拖拽 / 选择上传

    - 考勤数据视图：列表视图（支持多条件筛选）、日历视图（颜色标识考勤状态）

    - 导出中心：页面内模板设计器，支持模板修改、另存与实时预览

    - 考勤规则界面：考勤规则、假期管理配置页
	
	- 系统设置界面: 初始化数据，修改密码 二期有用户管理，角色管理，权限管理，连接数据相关，

2. 数据输出：

    - 本地 IndexedDB 存储的结构化考勤数据、历史结余数据

    - 可导出的 Excel 考勤凭证文件，包含可配置的考勤字段，作为财务发薪留档凭证

3. 追溯输出：单条考勤记录可关联查看原始考勤记录与 OA 单据数据

### 4. 关键约束与边界条件

#### 技术栈约束

- 前端框架：Vue 3（CDN 模式，无需构建工具）

- 样式库：TailwindCSS（CDN 模式）

- 本地数据库：IndexedDB，通过 Dexie.js 封装

- Excel 处理：SheetJS (xlsx) 库（或使用支持 OOXML 解析的库），需启用单元格样式读取能力，以支持读取单元格背景色信息，完成排班状态识别

- 部署：纯静态文件，浏览器直接打开`index.html`即可使用

- 网络适配：适配中国大陆网络环境，所有 CDN 依赖优先使用国内可稳定访问的镜像源，保障资源加载速度与稳定性

- 登录验证：前端形式验证，默认账号`admin` / `admin123`

#### 功能约束

1. **数据导入规则**

    - 自动识别文件：根据文件名关键词（考勤、请假、加班、出差、排班）及文件结构自动判定文件类型，无需用户手动指定

    - 数据匹配规则：以 "考勤号" 为第一唯一匹配键，无考勤号时降级为 "姓名 + 部门" 匹配

    - 原始数据约束：系统内不支持手动修改原始考勤数据，如需修正需用户修改源 Excel 后重新导入
	
	- 数据识别：看考勤数据日期定是哪个月的，不是现在是4月份导入的数据就是4月的，要看考勤打卡记录，是2026年2月，这个考勤数据就是2月的。
	
	- 字段识别：由于是多个系统的文件，字段是不定的，考勤里可能是姓名，请假单里的可能是申请人，要能自动匹配上。指纹打卡数据里有“考勤号”，其它的文件里是没有的。
	
	- 显示数据排列：优先考勤号，再到部门，再到名字。

2. **考勤规则引擎约束**

    - 排班识别：排班表为多 Sheet 结构，仅识别名称为 1 月 - 12 月的 Sheet，其余 Sheet 自动忽略，排班表是日历样式，不是常规的表格；排班的上班 / 休息判定规则：单元格背景色为白色的判定为上班日，其余背景色判定为休息日；同时支持公司统一节假日 / 调休日配置，可兼容非标准格式排班表

    - 容错规则：每月允许 2 次迟到，且累计迟到时长≤30 分钟的情况，豁免迟到判定，超出则按正常迟到计算

    - 结余计算：计算当月考勤时，需读取上上个月的结余数据（调休余额、加班余额），系统需保留历史月份数据

    - 异常判定：自动标记漏打卡、迟到、早退、旷工，需结合排班、假期、请假数据避免误判

3. **权限约束**

    - 本阶段仅保留超级管理员角色，可查看全公司所有数据，不做前端权限隔离

4. **导出功能约束**

    - 无需上传 Excel 模板，直接在页面内配置导出字段，支持实时预览

    - 预置默认导出模板，支持基于默认模板修改后另存为新模板

    - 默认导出字段：`序号 | 考勤号码 | 姓名 | 部门 | 日期 | 对应时段 | 上班时间 | 下班时间 | 签到时间 | 签退时间 | 迟到时间 | 早退时间 | 加班时间 | 出差时间 | 是否旷工`

5. **暂缓功能**：本阶段不开发多角色权限、后端服务、SQL 数据库、数据备份恢复、员工花名册管理、手动修正考勤数据功能，这些功能留待第二阶段实现

6. **使用流程约束**：严格遵循管理员初始化配置→批量导入源文件→系统自动计算→管理员核对数据→配置导出模板→导出凭证→本地数据归档的流程

## 第二阶段：企业增强版

### 1. 角色与上下文定义

你现在是一名全栈开发工程师，在第一阶段纯前端 MVP 的基础上，进行考勤系统的企业级增强开发，引入后端服务与云端存储，解决第一阶段本地存储的安全性、数据持久化问题，完善企业管理流程，满足企业更复杂的考勤管理需求。

### 2. 清晰的任务目标

为考勤系统引入后端服务与 SQL 数据库，实现多角色权限控制、数据云端备份恢复、员工全生命周期管理与手动考勤修正功能，提升系统的安全性、数据可靠性与管理灵活性。

### 3. 具体的输入输出格式

#### 输入

1. 管理员 / HR / 员工的不同角色操作输入

2. 员工花名册维护数据：员工新增、编辑、离职等信息、字段增加、有计算公式（工龄、生日等）

3. 手动考勤修正数据：管理员对考勤状态的手动调整记录，右键修改异常状态，单条加入：请假、加班、出差、漏打卡等信息

4. 数据备份 / 恢复输入：系统备份文件

#### 输出

1. 多角色交互界面：不同角色对应不同的数据查看 / 操作权限界面

2. 员工管理界面：员工花名册全生命周期管理界面

3. 数据备份恢复界面：支持 IndexedDB 整体数据的导出与导入恢复

4. 云端存储的结构化数据：加密存储的考勤、员工、权限数据

5. 操作日志：记录手动修正、数据修改等操作的追溯日志

### 4. 关键约束与边界条件

#### 技术栈扩展

- 在第一阶段前端技术栈基础上，引入后端服务，搭配 SQL 数据库实现数据持久化，有SQL数据库连接测试功能，连接到数据可添加库和初始化数据。

- 实现加密传输，保障数据传输安全

- 同样适配中国大陆网络环境，保障服务与资源的可访问性

#### 功能约束

1. **权限管理**：实现多角色权限隔离，不同角色可查看 / 操作对应权限范围内的数据

2. **数据安全**：实现后端 SQL 存储、加密传输，增加防清除缓存的机制，保障数据安全

3. **数据备份**：支持 IndexedDB 整体数据的导出备份，以及备份文件的导入恢复

4. **员工管理**：开发独立的员工花名册管理模块，支持员工信息的全生命周期管理，不再依赖导入数据生成员工信息

5. **手动修正**：支持管理员在界面上直接手动修正考勤记录，同时保留操作日志用于追溯

6. **密码安全**：实现密码的加密存储，替代第一阶段的前端明文形式验证
