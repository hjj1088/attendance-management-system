# 考勤管理系统 (Attendance Management System)

## 1. 项目简介
本系统是一款面向中小企业的轻量级考勤数据处理工具。**第一阶段（MVP）** 采用纯前端架构，利用浏览器本地存储实现无需部署的即时使用。核心目标是替代手工 Excel 计算，实现复杂考勤规则（排班、容错、结余）的自动化判定，为财务发薪提供准确凭证。

## 2. 开发阶段规划

### 第一阶段：核心功能版 (Current Phase)
*   **目标**：实现准确的考勤自动化计算与导出，满足财务归档需求。
*   **架构**：纯前端 (SPA) + IndexedDB 本地存储。
*   **重点**：复杂规则引擎、多源数据自动合并、页面内模板设计。
*   **权限**：仅超级管理员角色，查看全量数据。
*   **暂缓**：多角色权限隔离、后端服务、SQL 数据库、数据备份恢复、员工花名册管理、手动修正考勤数据。

### 第二阶段：企业增强版 (Future Phase)
*   **目标**：增强安全性、数据持久化与管理流程。
*   **架构**：引入后端服务 + SQL 数据库。
*   **重点**：多角色权限控制、数据云端备份/恢复、员工花名册全生命周期管理、手动修正考勤记录。

## 3. 第一阶段核心功能需求

### 3.1 数据导入与自动识别
1.  **批量导入**：
    *   支持一次性拖拽/选择多个 Excel 文件。
    *   **自动识别**：系统根据文件名关键词（如“打卡”、“请假”、“加班”、“出差”、“排班”）及文件内容结构，自动判定文件类型，无需用户手动指定。
2.  **支持文件类型**：
    *   指纹打卡机原始/汇总数据。
    *   OA 请假/加班/出差申请单。
    *   排班表（需识别班次及休息日）。
3.  **数据匹配**：
    *   以 **“考勤号”** 为第一唯一匹配键。
    *   若无考勤号，降级为“姓名 + 部门”匹配。
    *   只有考勤打卡数据里有 **“考勤号”**，其它文件“请假”、“加班”、“出差”、“漏打卡”是没有的，而且，姓名字段也不是姓名，是申请人或是别的。
4.  **数据原则**：
    *   源文件仅作为导入源和备查依据，**系统内不支持手动修改考勤原始数据**。
    *   如需修正，需修正源 Excel 后重新导入。

### 3.2 复杂考勤规则引擎 (核心)
1.  **排班与假期识别**：
    *   支持从导入的 Excel 排班表中识别班次。
    *   **公司统一调休**：支持设置公司级节假日/调休日（解决排班表中“不上班”的原因区分问题，如春节调休 vs 周末）。
    *   具备容错条件（能识别非标准格式的排班表）。
    *   排班表是多个Sheet文件，只有1月-12月是排班表，其它可以不管，排班表是日历样式的，不是列表样式，上班和休息是看日期单元格里的背景，休息是：白色，背景1，深度25%、35%，上班是白色。
2.  **容错规则 (Grace Period)**：
    *   **迟到豁免**：每月允许 **2 次** 迟到，且累计迟到时长 **≤ 30 分钟**，不计为迟到（视为正常）。
    *   超出上述范围则按正常迟到计算。
3.  **结余计算 (Carry-over)**：
    *   支持跨月数据关联。
    *   计算当月考勤时，需读取 **上上个月的结余数据**（如调休余额、加班余额），以判断当月考勤状态有效性。
    *   系统需保留历史月份数据以供查询结余。
4.  **异常判定**：
    *   自动标记：漏打卡、迟到、早退、旷工。
    *   结合排班、公司假期与请假数据，避免误判。

### 3.3 数据展示与查询
1.  **视图模式**：
    *   **列表视图**：支持按部门、人员、月份、状态筛选。
    *   **日历视图**：按月展示，颜色标识状态（绿 - 正常，黄 - 异常，红 - 缺勤）。
2.  **数据权限**：
    *   **第一阶段**：默认超级管理员登录，**查看全公司所有数据**，不进行前端权限隔离。
3.  **原始数据追溯**：
    *   点击任意考勤记录，可查看关联的原始打卡记录及 OA 单据。

### 3.4 导出中心 (页面内模板)
1.  **模板设计器**：
    *   **无需上传 Excel 模板**，直接在页面内设置导出字段。
    *   **实时预览**：设置模板时可预览导出效果。
    *   **模板管理**：系统预置一个默认模板，支持基于默认模板修改并“另存为”新模板。
2.  **默认导出字段**：
    *   `序号 | 考勤号码 | 姓名 | 部门 | 日期 | 对应时段 | 上班时间 | 下班时间 | 签到时间 | 签退时间 | 迟到时间 | 早退时间 | 加班时间 | 出差时间 | 是否旷工`
3.  **导出目的**：
    *   作为财务部发工资的凭证留档。
    *   导出的文件记录需可追溯（基于模板生成）。

### 3.5 系统设置
1.  **考勤规则配置**：
    *   上下班时间基准。
    *   迟到/早退阈值设置。
    *   容错规则参数配置（次数、时长）。
2.  **假期管理**：
    *   公司统一节假日/调休日设置。
    *   休息日配置。

### 3.6 技术栈约束
*   **前端框架**：Vue 3 (CDN 模式，无需构建)。
*   **样式库**：TailwindCSS (CDN 模式)。
*   **数据库**：IndexedDB (通过 Dexie.js 封装)。
*   **Excel 处理**：SheetJS (xlsx)。
*   **部署**：静态文件，浏览器直接打开 `index.html` 即可使用。
*   **默认账号**：`admin` / `admin123` (前端验证，仅形式)。

## 4. 暂缓功能清单 (列入第二阶段)
1.  **数据安全**：后端 SQL 存储、加密传输、防清除缓存机制。
2.  **数据备份**：IndexedDB 整体导出/恢复功能。
3.  **员工花名册**：独立的员工信息管理模块（目前依赖导入数据生成）。
4.  **手动修正**：在界面上直接修改考勤状态的功能。
5.  **高级安全**：多角色权限隔离、密码加密存储。

---

## 5. 使用流程 (第一阶段)
1.  **初始化**：管理员登录，设置考勤规则（含容错参数、公司假期）。
2.  **导入**：一次性拖拽上传上月所有相关 Excel 文件（系统自动识别分类）。
3.  **计算**：系统自动运行规则引擎，计算结余与状态。
4.  **核对**：管理员通过列表/日历视图核对异常（只读，源数据错误需重新导入）。
5.  **模板设置**：在导出中心确认或调整导出模板，预览效果。
6.  **导出**：生成考勤凭证 Excel 文件。
7.  **归档**：本地数据保留在浏览器中，供后续月份计算结余使用。
